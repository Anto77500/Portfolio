Architecture Réseau Segmentée Multi-VLANs

Conception, isolation logique et routage filtré d'un réseau d'entreprise sous Cisco Packet Tracer V8.

ContexteCabinet SecurAssur
SolutionCisco IOS v15 / Trunk 802.1Q
SécuritéAccess Control Lists (ACL)
1. Contexte & Problématique

Le cabinet d'assurances SecurAssur accueille quotidiennement des collaborateurs internes et des clients externes (invités) au sein de ses locaux. L'infrastructure d'origine ne disposait d'aucun cloisonnement : tous les périphériques partageaient le même domaine de diffusion (Flat Network).

Problématique identifiée : Un ordinateur invité, simplement connecté en Wi-Fi ou sur une prise Ethernet libre de la salle d'attente, pouvait analyser le trafic réseau local, découvrir les serveurs internes de l'entreprise et intercepter des données financières confidentielles ou injecter un malware sur le parc informatique.

2. Objectifs Techniques

Pour remédier à cette faille de sécurité majeure, le cahier des charges imposes les exigences suivantes :

  • Segmentation logique : Isolation complète du trafic réseau via la création de deux réseaux virtuels distincts : le VLAN 10 (Administration) et le VLAN 20 (Invités).
  • Routage Inter-VLAN : Implémentation d'une topologie Router-on-a-stick pour acheminer les flux autorisés vers la passerelle via une liaison partagée.
  • Contrôle des flux : Interdiction stricte aux hôtes du VLAN 20 d'initier une session de communication ou un scan vers les machines du VLAN 10.

4. Réalisation Technique

Étape 4.1 : Configuration des VLANs sur le Commutateur

Création des identifiants et assignation des interfaces d'accès (Postes de travail) sur le switch Cisco Catalyst 2960 :

Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Administration
Switch(config-vlan)# vlan 20
Switch(config-vlan)# name Invites
Switch(config-vlan)# exit
Switch(config)# interface fastEthernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

Étape 4.2 : Activation du Trunking et Routage Inter-VLAN

Configuration du port de liaison montante en mode Trunk 802.1Q pour transporter les trames tagguées de tous les VLANs vers le routeur :

Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# switchport mode trunk

Sur le routeur Cisco 1941, configuration des sous-interfaces logiques servant de passerelles d'infrastructure :

Router(config)# interface gigabitEthernet 0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.254 255.255.255.0
Router(config-subif)# exit
Router(config)# interface gigabitEthernet 0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.254 255.255.255.0

Étape 4.3 : Implémentation de la Politique de Sécurité (ACL)

Création d'une liste de contrôle d'accès standard pour bloquer le réseau des invités à destination de la zone d'administration sensible :

Router(config)# access-list 1 deny 192.168.20.0 0.0.0.255
Router(config)# access-list 1 permit any
Router(config)# interface gigabitEthernet 0/0.10
Router(config-subif)# ip access-group 1 out

5. Bilan & Compétences

Bilan opérationnel : L'infrastructure réseau du cabinet SecurAssur est désormais segmentée conformément aux exigences de sécurité. Les invités accèdent exclusivement à leur passerelle Internet sans pouvoir interférer ou scanner le sous-réseau interne contenant l'annuaire d'entreprise.
